## Bug
When running `pearls import-beads <file>`, the command reports "imported N issue(s)" but no `.md` files appear in the todos directory. The `memories.jsonl` file is written correctly, but the generated todo files are missing.

## Reproduction
```sh
WORK="$(mktemp -d)" && PI_TODO_PATH="$WORK/todos" npx tsx src/cli.ts import-beads /dev/stdin <<'JSONL'
{"id":"sldb-3l0","title":"Don't use docker for CDK","description":"Copy shared libs.","status":"closed","priority":0,"issue_type":"task","created_at":"2026-04-21T15:35:49Z"}
JSONL
ls "$WORK/todos/"  # empty — no .md files
```

## Impact
- Test suite `test/cli.sh` fails at the "search found imported issue" assertion (1/118 fails)
- `import-beads` is essentially non-functional for creating todos

## Discovered
Found while running `npm test` after the `extensions/todo.ts → extensions/pearls.ts` rename. The bug is pre-existing and unrelated to the rename.
